## 0. 定位判断(为什么是"大修可发表"而非"拒稿")

BiB 不要求广泛兴趣、变革性突破或湿实验验证;它重视**方法新颖性、基准严谨、可复现、软件可用、诚实报告**。
本稿在这些点上强(机制卡范式、预注册、去环化自审、透明度),因此短期只需补三类硬伤即可达投稿线:
**统计严谨(P0)→ 增量价值论证(P1)→ 可复用/可复现证据(P2)→ 润色(P3)**。

**投稿门槛判定:** 完成全部 P0 + P1-1 + P1-3 + P2-3 即达到可投稿的最低集合;P1-2/P2-1/P2-2 显著提升过审概率。

## 1. 任务分解(按优先级,含验收标准 Definition of Done)

### P0 — 统计严谨(必须,直接影响结论可信度)

**P0-1 单细胞相关的伪重复校正** ⭐最关键
- 现状:8,310 NK 细胞来自 9 个样本,H2–H5 的单细胞相关把细胞当独立观测,P 值被夸大。
- 动作:对所有单细胞层相关(H2、H3、H4、H5)改用二选一——
  (i) 线性混合模型,随机截距 = `sample_id`;或
  (ii) 逐样本计算 r,再随机效应 meta 合并。
- 落点:产出 `sst_axis_positive_control_recovery.tsv` 的校正版(新增列:校正后 r、95% CI、校正后 P、方法标注)。
- **DoD:** Table 2 与正文 §3.2 全部单细胞 P/r 替换为校正值;正文加一句方法说明;若 H3 效应臂校正后仍稳健,明确陈述"稳健于伪重复校正"。

**P0-2 计数深度/技术协方差控制**
- 动作:确认模块打分(mean z-score)在求相关前是否对 `total_counts`、`n_genes` 残差化;若未做,补做后重算 H2/H3。
- **DoD:** Methods §2.3 增加一句处理说明;附一句"H2/H3 在计数深度残差化后方向/显著性不变"(或如实报告变化)。

**P0-3 效应臂环化的更严零假设**
- 现状:通用 16 基因激活对照(偏 r=0.286)不足以排除细胞骨架-突触与效应颗粒的特异共调控。
- 动作:二选一或并用——
  (i) 对 scVI 潜因子(或前若干主成分)残差化后重算 protrusion~cytotoxicity;
  (ii) 在 NK 表达程序内置换两模块成员,构建经验零分布,报告实测 r 的经验 P。
- **DoD:** §3.2 增补一段"更严零假设下效应臂仍稳健(或减弱到 X)"的结果 + 一个补充图/表。

### P1 — 增量价值论证(决定"图/机制卡值不值")

**P1-1 与同类生物学方法基准比较** ⭐
- 现状:只比通用 ML 基线(XGBoost 等);BiB 会问"相对领域内既有工具的增益"。
- 动作:在相同折上,增加与 Scissor、CIBERSORTx/quanTIseq 反卷积驱动的轴打分对比(至少在 NK-state 分类或轴恢复其一)。
- **DoD:** Table 3 或新表加入同类方法行;正文明确 GNN/图嵌入相对它们的定量增益或对等结论。

**P1-2 靶点清单的平凡基线**
- 动作:构造"仅按机制卡成员资格排序基因"的平凡基线,与五维打分的 37 基因清单对比重合度与排序差异。
- 落点:`src/interpretation/split_target_lists.py` 旁增一个基线脚本 + 对比表。
- **DoD:** §3.5 加一句"五维打分相对平凡基线的增量为 X";若增量小,则全文一致改称"机制交集候选短名单"。

**P1-3 把"转录不代理物理拓扑"升为一级发现** ⭐
- 动作:将 H4/H5-protrusion 的符号相反(机器 mRNA↑ 而功能↓)从边界注脚,提为 Abstract/Key Points/Discussion 的核心结果之一,并明确它是"通向湿实验验证的桥梁"(指向项目二)。
- **DoD:** Abstract 与 Key Points 各出现一句该发现;Discussion §4.1 将其列为三大发现之一(已部分具备,强化措辞)。

### P2 — 可复用/可复现(BiB 重点)

**P2-1 第二张机制卡端到端结果**
- 动作:用 `configs/mechanism_cards/registry.yaml` 已提到的 adenosine 或 TGFβ 卡,跑通最小端到端结果(哪怕仅恢复图),作为可复用性证据。
- **DoD:** 补充材料给出第二张卡的最小结果 + 一段说明;**或**(退路)把摘要/要点的"引擎/可复用"改为"已在一张卡演示、多卡为设计意图"。

**P2-2 代码可复现证据**
- 动作:确认并补齐 README(安装+运行+合成数据模式)、环境锁定(requirements/conda)、关键模块单元测试(用 `.venv/Scripts/python.exe` 跑通 `src/common/synthetic_data.py` 合成路径)。
- **DoD:** 仓库根 README 可从零跑通合成模式;Data Availability 链接到可复现说明。

**P2-3 摘要/要点/结论一致性** ⭐
- 动作:消除"引擎(现在时贡献)"与结论"仍是设计愿景"的矛盾;靶点清单全文统一定位为假设生成短名单。
- **DoD:** 通读一遍,三处口径一致。

### P3 — 润色/排版

- **P3-1** 一张主线机制/流程示意图统领 14 阶段叙述(可用 nature-figure)。
- **P3-2** 参考文献 Vancouver 重排;标题精简;补 7 位作者 ORCID。
- **P3-3** 投稿前目视核对 `02_figures/` 面板与 `03_supplementary/` 完整性(本次评审未查图像/补充)。

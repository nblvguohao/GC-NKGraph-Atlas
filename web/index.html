What is a Mechanism Card?

A mechanism card is a YAML document that encodes one published wet-lab immune-evasion mechanism as a machine-readable recipe. Each card specifies:

🔬 Biology

  • Ordered mechanistic chain (cause → effect)
  • Cell types involved
  • Phenotype definition

🧬 Transcriptional Proxy

  • Gene modules per mechanism step
  • Expected direction per module
  • Cell-type attribution rules
  • Claim boundaries (what CANNOT be inferred)

📐 Graph Integration

  • New node types (e.g. tumor_serine_program)
  • New edge types (e.g. metabolic_crosstalk)
  • Construction rules with biological justification

✅ Validation

  • Positive control cohort
  • Pre-registered hypotheses
  • Recovery definition
  • Gold-standard gene list

The key innovation: the pipeline core never changes. To operationalize a new mechanism, you write a new card — not new code. The card formalizes what the mechanism IS, what transcriptomes CAN measure about it, and what they CANNOT.

How This Site Works

This playground loads the live mechanism cards from configs/mechanism_cards/. The same YAML files that drive the computational pipeline also power this interactive browser — demonstrating that the cards are genuinely machine-readable, not just documentation.
